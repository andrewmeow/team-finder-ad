# TeamFinder

Платформа, на которой разработчики, дизайнеры и другие специалисты смогут находить единомышленников для совместной работы над Pet-проектами. Зарегистрированные пользователи смогут публиковать идеи проектов, находить команду на проект и откликаться на опубликованные предложения.

## Основные возможности

- **Просмотр проектов** – лента актуальных проектов с фильтрацией по необходимым для проекта навыкам.
- **Регистрация и аутентификация** – вход по email, смена пароля.
- **Профиль пользователя** – аватар, контакты, список своих проектов.
- **Редактирование профиля** – изменение имени, фамилии, описания, GitHub, телефона.
- **Создание и редактирование проекта** – название, описание, статус.
- **Управление навыками проекта** – добавление/удаление необходимых технологий (с автодополнением).
- **Отклик на проект** – кнопка «Участвовать» / «Отказаться от участия».
- **Завершение проекта** – доступно только владельцу.
- **Список всех пользователей** – с пагинацией и фильтрацией по навыкам.
- **Администрирование** – управление пользователями и проектами через встроенную Django‑админку.

## Технологический стек

- **Backend:** Python 3.10, Django 5.2, Gunicorn
- **Database:** PostgreSQL 16
- **Frontend:** HTML, CSS, JavaScript
- **Web‑сервер:** Nginx
- **Контейнеризация:** Docker, Docker Compose
- **CI/CD:** GitHub Actions

## Запуск с Docker Compose

**Предварительные требования:**
- Установленный [Docker](https://docs.docker.com/get-docker/) и Docker Compose.
- Свободный порт 80 (или измените маппинг портов в `docker-compose.yml`).

### 1. Клонирование репозитория

```bash
git clone git@github.com:<your_nickname>/team-finder-ad.git
cd team-finder-ad
```

### 2. Настройка переменных окружения

Файл `.env` содержит конфиденциальные настройки проекта — ключ Django, параметры БД и другие переменные. Создайте файл `.env` в корне проекта на основе `.env_example`:

```bash
cp .env_example .env
```

После этого откройте `.env` и укажите свои значения.  

| Переменная               | Назначение                                                                                                                                                 |
|--------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **DJANGO_SECRET_KEY**    | Секретный ключ Django, используемый для подписи cookie и токенов. Можно сгенерировать при помощи `get_random_secret_key` из `django.core.management.utils` |
| **DJANGO_DEBUG**         | Режим отладки.                                                                                                                                             |
| **DJANGO_ALLOWED_HOSTS** | Список доменов, разделённых пробелами.                                                                                                                     |
| **POSTGRES_DB**          | Имя базы данных PostgreSQL, которую будет использовать Django.                                                                                             |
| **POSTGRES_USER**        | Имя пользователя PostgreSQL.                                                                                                                               |
| **POSTGRES_PASSWORD**    | Пароль пользователя PostgreSQL.                                                                                                                            |
| **POSTGRES_HOST**        | Адрес сервера БД. В случае локального развертывания - localhost.                                                                                           |
| **POSTGRES_PORT**        | Порт подключения к БД (по умолчанию `5432`).                                                                                                               
---

### 3. Запуск контейнеров

```bash
docker-compose up -d
```
`-d` значит `detach`, то есть контейнер продолжит работать в фоне. Чтобы его остановить, надо будет ввести

```bash
docker compose down
```
Эта команда:
- соберёт образ backend‑приложения (или скачает готовый, если указан `image`),
- запустит сервисы PostgreSQL, Django‑приложение и Nginx,
- автоматически применит миграции,
- создаст тома для сохранения данных.

### 4. Создание суперпользователя

```bash
docker-compose exec web python manage.py createsuperuser
```

### 5. Заполнение тестовыми данными (опционально)

Для контейнера Docker:
```bash
docker-compose exec web python manage.py filldb
```

Команда `filldb` создаёт:
- Пользователя `alice@example.com` (пароль `strongpassword1`) с проектом "Сайт портфолио".
- Пользователя `bob@example.com` (пароль `strongpassword2`) с проектом "API для блога".
- Боб присоединён к проекту Алисы в качестве участника.

### 6. Доступ к приложению

- Основной сайт: [http://localhost](http://localhost)
- Админ‑панель: [http://localhost/admin](http://localhost/admin)

## Локальная разработка без Docker

**Предварительные требования:**
- Python 3.10+
- Установленный и настроенный PostgreSQL (можно запустить отдельный контейнер только с PostgreSQL, используя `docker-compose up -d db`)

### 1. Клонирование репозитория и виртуальное окружение

```bash
git clone git@github.com:<your_nickname>/team-finder-ad.git
cd team-finder-ad
```

Перед началом работы необходимо создать и активировать виртуальное окружение Python.  

1. **Создайте виртуальное окружение (в папке проекта):**
   ```bash
   python3 -m venv venv
   ```

   После этого появится папка `venv`, где будут храниться зависимости проекта.

2. **Активируйте окружение:**

    - **Windows (PowerShell):**
      ```bash
      venv\Scripts\Activate.ps1
      ```
    - **Windows (cmd):**
      ```bash
      venv\Scripts\activate
      ```
    - **Linux/Mac:**
      ```bash
      source venv/bin/activate
      ```

3. **Установите зависимости из `requirements.txt`:**
   ```bash
   pip install -r requirements.txt
   ```

   После установки в окружении будут доступны все нужные библиотеки Django-проекта.

### 2. Настройка `.env`

Скопируйте `.env_example` в `.env` и заполните, как описано [выше](#2-настройка-переменных-окружения). Для локального запуска укажите `POSTGRES_HOST=localhost`.

### 3. Запуск PostgreSQL

Убедитесь, что сервер PostgreSQL работает по адресу `localhost:5432` и существует база данных с именем из `POSTGRES_DB`. При необходимости создайте её:

```sql
CREATE DATABASE team_finder;
```

Или используйте отдельный PostgreSQL‑контейнер:

```bash
docker run --name teamfinder-postgres -e POSTGRES_DB=team_finder -e POSTGRES_USER=team_finder_user -e POSTGRES_PASSWORD=пароль -p 5432:5432 -d postgres:16
```

### 4. Применение миграций

```bash
python manage.py migrate
```

### 5. Тестовые данные (опционально)

```bash
python manage.py filldb
```

### 6. Запуск сервера разработки

```bash
python manage.py runserver
```

Сайт будет доступен по адресу [http://localhost](http://localhost).
